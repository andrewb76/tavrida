# 📖 Oracle — глоссарий

> **Статус:** draft · **Версия:** 0.1  
> Читать **первым** перед остальными документами Oracle.

| Термин | По-русски | Что значит |
|--------|-----------|------------|
| **Oracle / Оракул** | Оракул | Служебный калькулятор прогноза дохода для founder/admin. Не трогает реальные деньги пользователей. |
| **Assumption** | Допущение | Число, которое вы **предполагаете** (например «40 регистраций в месяц»). Живёт только в симуляции. |
| **Scenario** | Сценарий | Набор допущений с именем: «базовый», «оптимистичный», «пессимистичный». |
| **Preset** | Пресет | Готовый сценарий из конфига с множителями (×1.5 регистраций и т.д.). |
| **Growth model** | Модель роста | Как меняется приток пользователей во времени: ровно, ускорение, S-кривая. |
| **Cohort** | Когорта | Группа людей, пришедших в один период; от неё считаем удержание и деньги. |
| **Plan mix** | Распределение по тарифам | Какая доля новичков садится на Free / Basic / Pro. |
| **Churn** | Отток | Доля платных подписок, которые **не продлили** и вернулись на Free. |
| **MRR** | Ежемесячная выручка подписок | Monthly Recurring Revenue — деньги от Basic/Pro **каждый месяц**. |
| **Gross** | Валовый доход | Всё, что платформа **заработала** до вычета расходов и реферала. |
| **Net** | Чистый результат | Gross − реферал − налоги/комиссии − фиксированные затраты. |
| **One-time charge** | Разовый платёж | Списание один раз: продвижение лота 200₽, реакция на форуме 50₽. |
| **Attach rate** | Доля подключения | Какой % лотов получает платную опцию (например 10% лотов с promotion). |
| **GMV** | Оборот сделок | Сумма сделок **между участниками** — **не** доход платформы (справочно в Oracle). |
| **Burn** | Месячный burn | Сколько ₽/мес уходит на содержание (хостинг, зарплаты, сервисы). |
| **Break-even month** | Месяц окупаемости | Первый месяц, когда накопленный чистый результат стал неотрицательным. |
| **Referral outflow** | Расход на реферал | Выплаты пригласившим по программе `referral-rewards`. |
| **Depth** | Глубина дерева | Уровень в цепочке приглашений: 1 = прямой друг, 2 = друг друга. |
| **Engine** | Движок расчёта | Пакет `@tavrida/monetization-engine` — формулы в коде. |
| **Defaults YAML** | Файл диапазонов | `config/oracle.defaults.yaml` — min/max/default ползунков. |

## Чего Oracle **не** делает

- Не списывает деньги с кошельков.
- Не меняет цены в production (`financial-policy`, `billing`).
- Не предсказывает погоду и не заменяет бухгалтерию — это **модель «что если»**.
