## 🔍 Анализ существующих решений

| Платформа | Архитектура | Плюсы | Минусы | Применимо к Tavrida Lot? |
|-----------|-------------|-------|--------|--------------------------|
| **Discourse** | Node.js + PostgreSQL + Redis | Современный, REST API, встроенный спам-фильтр, реакции, уведомления | Много функций, которые не нужны (чат, теги, модерация), «тяжёлый» | ❌ Перегружен, сложно интегрировать |
| **Flarum** | PHP + MySQL | Легковесный, расширяемый, простой API | PHP (не Node.js), малая экосистема | ⚠️ Можно, но фреймворк не совпадает с остальными сервисами |
| **Vanilla Forums** | PHP/Node.js | Гибкий, встроенный SSO, API | Старый интерфейс, не очень активное развитие | ⚠️ Рассматриваем, если нужен «из коробки» |
| **Reddit-style** (SO, Hacker News, Lobsters) | Уникальная модель голосования, лента по популярности | Очень высокая вовлечённость, виральность | Не подходит для локального/торгового форума | ❌ |
| **Независимый сервис (NestJS)** | ✅ | Полный контроль, интеграция с `rating`, `billing`, `user-profile` | Нужно писать всё с нуля | ✅ **Идеально!** |

> ✅ **Вывод:** для **Tavrida Lot** — **свой форум-сервис** на NestJS с глубокой интеграцией:
>
> - `rating` → голоса за посты/комментарии = карма
> - `billing` → премиум-темы, уведомления, аналитика
> - `financial-policy` → лимиты постов, тегов, размера
> - `user-profile` → профиль, репутация, заметки

## 🌳 Ветки комментариев

> **Ключевое правило:** пользователь может ответить **на тему** или **на любой комментарий** в этой теме — вложенность **не ограничена одним уровнем** (ограничение только тарифом `forum.author.03thread.depthMax`, когда включено в plan-config).

### Модель

```text
Topic
├── Comment A          parentId = null
│   ├── Comment B      parentId = A
│   │   └── Comment C  parentId = B
│   └── Comment D      parentId = A
└── Comment E          parentId = null
```

- **Topic** — корневое сообщение (заголовок + тело); к нему нельзя «ответить» через `parentId` (только новый comment с `parentId: null`).
- **Comment** — узел ветки; прямой родитель — либо тема (`parentId: null`), либо другой comment (`parentId: <uuid>`).
- **Closure table** (`comment_closure`) хранит все пары предок–потомок для быстрых запросов поддерева, promote, scoped-модерации.

### API (BFF)

| Действие | Тело POST | Результат |
|----------|-----------|-----------|
| Комментарий к теме | `{ "body": "…" }` | `parentId: null` |
| Ответ на comment | `{ "body": "…", "parentId": "<commentId>" }` | новый узел под выбранным comment |

Список комментариев — **плоский** массив с полем `parentId`; UI собирает дерево (рекурсивный `CommentNode`, [W06](../../11-ux-ui/wireframes/forum.md)).

### Тарифы (plan-config, не scaffold)

| Возможность | Free | Basic | Pro | Ключ |
|-------------|------|-------|-----|------|
| Вложенные ответы (ответ на comment) | ❌ | ✅ | ✅ | `forum.author.10reply.nestedEnabled` |
| Макс. глубина ветки | 2 | 5 | ∞ | `forum.author.03thread.depthMax` |

На Free допустимы только корневые комментарии к теме (`parentId: null`). Ответ на comment — 403, если `nestedEnabled = false` или превышен `depthMax`.

### 🔹 Хранение реакций

```ts
@Entity({ schema: 'forum', name: 'reaction' })
export class Reaction {
  @PrimaryColumn('uuid')
  contentId: string // topicId или commentId

  @Column('enum', { enum: ['topic', 'comment'] })
  contentType: string

  @PrimaryColumn('uuid')
  userId: string

  @Column('varchar', { length: 20 })
  emojiKey: string

  @CreateDateColumn()
  createdAt: Date
}
```

## 🧠 Интеграция с другими сервисами

### 🔹 `rating`

- При создании поста — создаётся `rating` для автора (как в Reddit/Habr)
- Голоса за посты/комменты влияют на `verifiedSales` (например, если пост «рекомендация по покупке» — это «транзакция»)
- Карма влияет на лимиты: при низкой карме — лимит постов/комментов, проверка модерацией

### 🔹 `billing` / `financial-policy`

Pro-возможности форума и лимиты — через ключи `forum.*` в [PLATFORM-REGISTRY.md](../../PLATFORM-REGISTRY.md).  
Платные реакции — раздел «Разовые платежи» (`forum.reaction.*`).

## 🧩 Архитектура структур

| Вариант | Плюсы | Минусы | Выбор |
|---------|-------|--------|-------|
| **Flat structure (все в одной таблице)** | Просто | Сложные запросы для древовидных ответов | ❌ |
| **Nested set model** | Быстрые поддеревья | Сложные INSERT/UPDATE | ⚠️ |
| **Closure table (отдельная таблица родство-потомство)** | Быстрый поиск, простая вставка | Дублирование данных | ✅ |
| **Materialized path** | Баланс | Нужна валидация | ✅ |

> ✅ **Рекомендация**: `closure table` для ответов + `flat` для тем.

## 🔐 Безопасность и модерация

### 👮 Роли модератора форума

Два уровня модерации (назначает **администратор**; админ управляет всеми модераторами — назначение и снятие):

| Роль | Формат | Область действия |
|------|--------|------------------|
| **Модератор (главный)** | `Модератор` — без типа и ID | Любой объект форума: категории, топики, комментарии |
| **Модератор (областной)** | `Модератор:<ТипОбъекта:ID>` | Указанный объект **и все вложенные** |

Поддерживаемые `ТипОбъекта`:

| Тип | Пример | Вложенные объекты |
|-----|--------|-------------------|
| `category` | `Модератор:category:uuid` | все топики и комментарии в категории (включая подкатегории) |
| `topic` | `Модератор:topic:uuid` | топик и все комментарии в ветке |
| `comment` | `Модератор:comment:uuid` | комментарий и все ответы в его поддереве |

**Права модератора** на объекты в своей области (и вложенные): скрытие/удаление, pin, блокировка автора, просмотр жалоб, **выделение комментария в топик** (см. ниже).  
Главный модератор наследует те же права на весь форум. Администратор — полный доступ + управление назначениями модераторов.

Технически: [moderator-mapping.md](../../../09-security/moderator-mapping.md) · [keto-schema.md](../../../09-security/keto-schema.md) · [roles.md](../../../01-goal/roles.md).

### 🔀 Выделение комментария в топик (`promote comment → topic`)

Модератор (главный или с правом на объект/предка) может превратить **любой комментарий** в отдельный топик.

**До:**

```text
Topic A
├── Comment 1
├── Comment 2          ← выбран модератором
│   ├── Reply 2.1
│   └── Reply 2.2
└── Comment 3
```

**После:**

```text
Topic A                          Topic B (новый, равноправный с A)
├── Comment 1                    └── (текст бывшего Comment 2)
├── Comment 2  → ссылка на Topic B
├── Reply 2.1    → родитель = Topic B
├── Reply 2.2    → родитель = Topic B
└── Comment 3
```

**Алгоритм:**

1. Создаётся **новый топик** (`Topic B`) с текстом выбранного комментария (заголовок — из первой строки или задаёт модератор).
2. У **напрямую подчинённых** комментариев (дети выбранного) меняется родитель на `Topic B`.
3. У **выбранного комментария** добавляется ссылка на `Topic B` (он остаётся в старом топике как маркер ветвления).
4. **Старый и новый топики равнозначны** — оба продолжают жизнь независимо, обрастают новыми комментариями.

**API (draft):** `POST /internal/v1/forum/comments/{commentId}/promote-to-topic`  
**Событие:** `forum.comment_promoted_to_topic` → [event-catalog](../../../03-architecture/event-catalog.md).

**Модель данных (draft):**

```ts
// У комментария после promote
@Column({ type: 'uuid', nullable: true })
promotedTopicId: string | null  // ссылка на новый топик

// У дочерних комментариев — смена родителя
@Column({ type: 'uuid' })
parentId: string  // был commentId, стал topicId для прямых детей
```

Оба топика (`sourceTopicId`, `newTopicId`) остаются в одной категории, без иерархии «родитель-потомок» между собой.

**Требования:** атомарная транзакция; audit trail; доступ — [moderator-mapping.md](../../../09-security/moderator-mapping.md).

### 🛡️ Прочие меры

| Функция | Применение |
|--------|------------|
| **Автофильтр матов** | `forum.bannedWordsList` (settings), проверка при создании topic/comment |
| **Rate-limiting** | `forum.postsPerDay` (financial-policy) |
| **Анонимность** | `authorId = null` + `signedBy: UUID`, не виден другим |
| **Жалобы** | `forum.content_reported` → [notifications](../../notifications/README.md) |
| **Автомодерация** | `rating < 0.3` → topic/comment проходят ручную модерацию |

**Назначение модераторов:** только admin — см. [moderator-mapping.md](../../../09-security/moderator-mapping.md).
