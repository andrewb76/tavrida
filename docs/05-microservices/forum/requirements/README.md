## 🔍 Анализ существующих решений

| Платформа | Архитектура | Плюсы | Минусы | Применимо к Tavrida Lot? |
|-----------|-------------|-------|--------|--------------------------|
| **Discourse** | Node.js + PostgreSQL + Redis | Современный, REST API, встроенный спам-фильтр, реакции, уведомления | Много функций, которые не нужны (чат, теги, модерация), «тяжёлый» | ❌ Перегружен, сложно интегрировать |
| **Flarum** | PHP + MySQL | Легковесный, расширяемый, простой API | PHP (не Node.js), малая экосистема | ⚠️ Можно, но фреймворк не совпадает с остальными сервисами |
| **Vanilla Forums** | PHP/Node.js | Гибкий, встроенный SSO, API | Старый интерфейс, не очень активное развитие | ⚠️ Рассматриваем, если нужен «из коробки» |
| **Reddit-style** (SO, Hacker News, Lobsters) | Уникальная модель голосования, лента по популярности | Очень высокая вовлечённость, виральность | Не подходит для локального/торгового форума | ❌ |
| **Независимый сервис (NestJS)** | ✅ | Полный контроль, интеграция с `rating`, `billing`, `user-profile` | Нужно писать всё с нуля | ✅ **Идеально!** |

> ✅ **Вывод:** для **Tavrida Lot** — **свой форум-сервис** на NestJS с глубокой интеграцией:
>
> - `rating` → голоса за посты/комментарии = карма
> - `billing` → премиум-темы, уведомления, аналитика
> - `financial-policy` → лимиты постов, тегов, размера
> - `user-profile` → профиль, репутация, заметки

### 🔹 Хранение реакций

```ts
@Entity()
export class Reaction {
  @PrimaryColumn('uuid')
  postId: string // ID поста или темы

  @PrimaryColumn('uuid')
  userId: string

  @Column('varchar', { length: 20 })
  emojiKey: string // 'like', 'laugh', 'rocket'...

  @CreateDateColumn()
  createdAt: Date
}
```

### 🔹 API

#### `GET /api/v1/reactions?postId=xxx`

```json
{
  "postId": "xxx",
  "reactions": [
    { "emoji": "👍", "count": 12, "users": ["user1", "user2"] },
    { "emoji": "❤️", "count": 3, "users": [] }
  ]
}
```

#### `POST /api/v1/reactions`

```http
POST /api/v1/reactions
Authorization: Bearer {token}
```

```json
{
  "postId": "post-uuid",
  "emojiKey": "laugh"
}
```

**Ответ:**

- `200 OK` + `reactionId` — если уже ставил, обновляет
- `403 Forbidden` — если пробует использовать Pro-реакцию без подписки

## 🧠 Интеграция с другими сервисами

### 🔹 `rating`

- При создании поста — создаётся `rating` для автора (как в Reddit/Habr)
- Голоса за посты/комменты влияют на `verifiedSales` (например, если пост «рекомендация по покупке» — это «транзакция»)
- Карма влияет на лимиты: при низкой карме — лимит постов/комментов, проверка модерацией

### 🔹 `billing` / `financial-policy`

Pro-возможности форума и лимиты — через ключи `forum.*` в [PLATFORM-REGISTRY.md](../../PLATFORM-REGISTRY.md).  
Платные реакции — раздел «Разовые платежи» (`forum.reaction.*`).

## 🧩 Архитектура структур

| Вариант | Плюсы | Минусы | Выбор |
|---------|-------|--------|-------|
| **Flat structure (все в одной таблице)** | Просто | Сложные запросы для древовидных ответов | ❌ |
| **Nested set model** | Быстрые поддеревья | Сложные INSERT/UPDATE | ⚠️ |
| **Closure table (отдельная таблица родство-потомство)** | Быстрый поиск, простая вставка | Дублирование данных | ✅ |
| **Materialized path** | Баланс | Нужна валидация | ✅ |

> ✅ **Рекомендация**: `closure table` для ответов + `flat` для тем.

## 🔐 Безопасность и модерация

| Функция | Применение |
|--------|------------|
| **Автофильтр матов** | `forum.bannedWordsList` (settings), проверка при `POST /forum/posts` |
| **Rate-limiting** | `forum.postsPerDay` (financial-policy) |
| **Анонимность** | `authorId = null` + `signedBy: UUID`, не виден другим |
| **Жалобы** | Событие `forum.post_reported` → [notifications](../../notifications/README.md) |
| **Автомодерация** | `rating < 0.3` → посты проходят ручную модерацию |
