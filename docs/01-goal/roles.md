## 🎯 Зачем этот документ

На Tavrida Lot одновременно действуют **несколько «слоёв»**, которые часто путают:

| Слой | Что это | Пример |
|------|---------|--------|
| **Роль (identity)** | Кто вы в системе с точки зрения прав | пользователь, модератор, **эксперт**, админ |
| **Тариф (plan)** | Пакет лимитов и фич | Free, Basic, Pro |
| **Контекст сделки** | Временная роль в конкретном аукционе | продавец, покупатель, участник торгов |

**Тариф — не роль.** Пользователь с Pro остаётся «пользователем», но получает больше лимитов.  
**Продавец — не роль в системе.** Любой пользователь может выставить лот; в конкретном аукционе он становится продавцом.

---

## 🧑‍🤝‍🧑 Роли (identity)

### Гость (Guest)

**Кто:** не вошёл в аккаунт.

| Может | Не может |
|-------|----------|
| Смотреть каталог лотов (публичные) | Делать ставки |
| Читать открытые темы форума | Создавать темы и комментарии |
| Видеть профили (публичная часть) | Пополнять баланс, подписки |
| | Видеть приватные заметки |

---

### Пользователь (User)

**Кто:** зарегистрированный участник (базовая роль после входа через Logto).

**По умолчанию все новые аккаунты — User + тариф Free.**

| Область | Возможности |
|---------|-------------|
| **Аукционы** | Создавать лоты и делать ставки в рамках [лимитов тарифа](../05-microservices/PLATFORM-REGISTRY.md) |
| **Форум** | Темы, комментарии, реакции — в рамках тарифа |
| **Профиль** | Редактировать свой профиль, приватные заметки о других |
| **Деньги** | Баланс, история, подписка, разовые платежи |
| **Отзывы** | Оставлять feedback после сделок |
| **Подписки** | Подписаться на категории/аукционы |
| **Marketplace** | Заказывать услуги; публиковать услуги — если `listingsMax > 0` |

**Ограничения (наложенные поверх роли):**

- низкий рейтинг или много неоценённых сделок → лимиты от [rating](../05-microservices/rating/README.md)
- временный бан → блокировка ставок и постов
- недостаточный баланс → нет платных операций

---

### Модератор (Moderator)

**Кто:** доверенный участник или сотрудник. Назначается **админом**.  
**Модель уточняется** — см. [keto-schema.md](../09-security/keto-schema.md).

| Область | Может |
|---------|-------|
| **Форум** | Жалобы, скрытие/удаление постов, pin тем, ограничение пользователя |
| **Аукционы** | Просмотр жалоб на лот, скрытие/восстановление лота, блокировка ставок нарушителя на лоте |
| **Общее** | Не меняет баланс, тарифы, settings; не назначает admin |

| Не может |
|----------|
| Admin API (`/api/v1/admin/*`) |
| Прямое изменение баланса / подписок |
| Регистрация параметров platform |

> Споры по сделкам — через отзывы и рейтинг; модератор вмешивается при явных нарушениях правил.

---

### Эксперт (Expert)

**Кто:** специалист (реставратор, нумизмат, оценщик). Назначается **только админом**.

| Может | Не может |
|-------|----------|
| Добавлять **экспертную оценку** к лоту аукциона | Назначать других экспертов |
| Несколько экспертов — несколько оценок на один лот | Менять цену/ставки на лоте |
| Портфолио услуг в marketplace (если тариф позволяет listings) | Обходить лимиты тарифа |

**Эффект экспертизы:** оценка видна на странице лота, повышает «значимость» (бейдж, сортировка).  
Технически: [auction expert-appraisals](../05-microservices/auction/README.md), Keto `platform#expert`.

> Expert **не заменяет** тариф: на Free по-прежнему `marketplace.listingsMax = 0`, если не изменено в реестре.

---

### Администратор (Admin)

**Кто:** оператор платформы, технический или business-админ.

| Область | Возможности |
|---------|-------------|
| **Тарифы и лимиты** | Регистрация параметров, значения Free/Basic/Pro ([financial-policy](../05-microservices/financial-policy/README.md)) |
| **Настройки** | Скалярные параметры платформы ([settings](../05-microservices/settings/README.md), [реестр](../05-microservices/PLATFORM-REGISTRY.md)) |
| **Пользователи** | Назначение модераторов и **экспертов**, перманентный бан, просмотр audit |
| **Контент** | Полная модерация, скрытие лотов |
| **Планы** | Цены подписок, активация/отмена в исключительных случаях |

Доступ: `/api/v1/admin/*` через BFF + проверка роли в **Ory Keto**.

---

### Система (Service)

**Кто:** внутренние микросервисы (не люди).

- Вызовы `/internal/v1/*` между сервисами
- Публикация и потребление событий RabbitMQ
- Не проходят через публичный BFF

## 📦 Тарифы (не роли, но влияют на права)

Тариф задаёт **лимиты и фичи**, не меняя identity-роль. Подробные значения — в [реестре переменных](../05-microservices/PLATFORM-REGISTRY.md).

| Тариф | Для кого | Суть |
|-------|----------|------|
| **Free** | Начало | Мало аукционов, базовый форум, английский аукцион |
| **Basic** | Активные участники | Больше лимитов, голландский аукцион, вложенные ответы |
| **Pro** | Продавцы, лидеры мнений | Почти без лимитов, продвижение, аналитика, расширенный форум |

Пользователь **всегда** имеет роль User; тариф хранится в `UserSubscription` (financial-policy).

## 🔄 Контекстные роли в аукционе

В рамках **одного лота** у человека может быть разная «шапка»:

| Контекст | Кто | Права |
|----------|-----|-------|
| **Продавец (seller)** | Создал лот | Редактировать до первой ставки, завершить досрочно (если правила позволяют), получить feedback |
| **Покупатель (buyer)** | Победил в торгах | Подтвердить получение, оставить отзыв |
| **Участник (bidder)** | Делал ставки, но не победил | Только история участия |
| **Наблюдатель** | Смотрит без ставок | Просмотр, подписка на уведомления |

Контекст **не назначается админом** — определяется автоматически по данным аукциона.

| Публикация услуги (marketplace) | ❌ | ✅¹ | ✅¹ | ✅¹ | ✅ |
| Экспертная оценка лота | ❌ | ❌ | ❌ | ✅ | ✅ |
| Жалоба на пост | ❌ | ✅ | ✅ | ✅ | ✅ |
| Модерация форума | ❌ | ❌ | ✅ | ❌ | ✅ |

| Изменение settings | ❌ | ❌ | ❌ | ❌ | ✅ |

| Назначение модератора / эксперта | ❌ | ❌ | ❌ | ❌ | ✅ |

## 🛡️ Техническая реализация

- **Logto** — JWT, `sub` = userId
- **Ory Keto** — роли (`member`, `moderator`, `expert`, `admin`) и ownership: **[keto-schema.md](../09-security/keto-schema.md)**
- **financial-policy** — лимиты тарифов (не Keto)
- **rating** — баны и штрафы (не Keto)
